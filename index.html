The read

Modern Trials is past proof-of-concept. The next wall is throughput.

The ARPA-H Health Innovation Network listing, the InterSystems collaboration, and Blackford Middleton on the advisory bench say the same thing. Modern Trials is past proof-of-concept. The next wall is throughput.

Every new hospital drags a FHIR integration behind it. Every new sponsor asks questions about AI extraction accuracy that a procurement team will not take "trust us" for an answer on. A ten-person company cannot carry both.

01

Integration throughput ceiling

Every new health system and every new sponsor protocol triggers a bespoke FHIR and EHR extraction effort. That work caps revenue before the sales team even runs out of pipeline.

02

AI claim exposure with sponsors

Pharma and CRO procurement teams will want auditable evidence that AI-driven extraction is not over-counting eligible patients. Without a governance layer, a technical review stalls the deal.

03

Two-buyer GTM tax

Provider orgs and sponsors are different buying motions, different price points, different success metrics. Pulling in both directions slows close rates on either side.

The Eduba frame

A 60/30/10 split on most healthtech stacks.

Sixty percent of what feels like AI work is traditional code and schema work, done once and reused. Thirty percent is rule-based logic that sits above the code. Only the final ten percent is a genuine model decision, where the cost of a wrong call earns the LLM tax.

For Modern Trials, the integration playbook (EMR parsing, FHIR mapping, consent workflow, site onboarding) is code and rules. Eligibility screening against a trial protocol is where real AI earns its place. The question is whether the current stack separates those layers, or whether model calls are quietly carrying logic that Postgres would do faster, cheaper, and with a cleaner audit trail.

Computational orchestration

Deciding which layer each piece of the problem belongs on.

60%
Traditional code
EMR parsing, FHIR mapping, consent workflow, site onboarding, scheduling. Done once. Reused every deployment.
30%
Rule-based logic
Protocol decision trees, outreach cadences, document routing, audit trail policies. Postgres and state machines, not LLMs.
10%
Genuine model work
Eligibility screening across messy free-text notes. The place where a wrong answer earns the cost of a model call.
